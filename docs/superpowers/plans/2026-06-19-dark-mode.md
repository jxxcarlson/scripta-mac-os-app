# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a whole-app dark mode — a manual toolbar toggle (persisted across launches) that switches chrome, editor, and rendered preview between light and dark.

**Architecture:** A CSS custom-property palette in `index.html` (`:root` light values + a `[data-theme="dark"]` override). Elm's `view` puts `data-theme` on its outermost element and paints that element's background from the palette; CSS variables inherit down the tree and pierce shadow DOM, so the one attribute drives both the chrome (Elm inline styles referencing `var(--…)`) and the CodeMirror editor (already reads `--cm-*`). The Scripta preview is already driven by the existing `model.isLight → Scripta.Dark`. Persistence mirrors the existing `readerMode` pattern (Flags + `saveX` port + localStorage).

**Tech Stack:** Elm 0.19.1, `elm-explorations/test`, Tauri 2 (no Rust changes), CSS custom properties.

Spec: `docs/superpowers/specs/2026-06-19-dark-mode-design.md`
